# TaskFlow Codebase Summary

This document provides a high-level overview of the TaskFlow application architecture, features, and configuration.

## 1. Project Structure

- `android/` - Capacitor Android native project files.
- `ios/` - Capacitor iOS native project files.
- `dist/` - Production build output directory (generated by Vite).
- `node_modules/` - NPM dependencies.
- `public/` - Static assets served natively (not bundled by Vite).
- `app.js` - Main application logic, UI rendering, event handling, and DOM interactions.
- `style.css` - Application styling, design system tokens, and responsive layout rules (desktop & mobile).
- `index.html` - Main HTML structure, containing both the authentication UI and the main application dashboard templates.
- `firebase-config.js` - Firebase SDK initialization and exported instances (auth, db).
- `notifications.js` - Capacitor Local Notifications wrapper for scheduling and managing task reminders.
- `sanitize.js` - DOMPurify utility wrapper for preventing XSS attacks when rendering user input.
- `package.json` / `package-lock.json` - NPM dependencies and script definitions.
- `vite.config.js` - Vite bundler configuration.
- `capacitor.config.json` - Capacitor configuration for mobile native builds.
- `firebase.json` - Firebase Hosting and project rules configuration.
- `firestore.rules` - Firestore security rules.
- `firestore.indexes.json` - Firestore composite index definitions.
- `.env` - Environment variables configuration.
- `.firebaserc` - Firebase CLI project alias mapping.
- `.gitignore` - Git ignored files configuration.

## 2. Tech Stack

- **Frontend Core:** Vanilla JavaScript, HTML5, CSS3
- **Bundler:** Vite (`^8.0.9`)
- **Backend/Database:** Firebase (`^12.12.1`) - Auth, Firestore
- **Mobile Cross-platform Framework:** Capacitor (`^8.3.1`)
- **Utility Libraries:**
  - `dompurify` (`^3.4.1`) - For sanitizing HTML to prevent XSS.
  - `flatpickr` (`^4.6.13`) - For date/time selection UI.

## 3. Features List

- **Authentication:** Google Sign-In and Email/Password authentication.
- **Task Management:** Create, Read, Update, and Delete (CRUD) operations for tasks.
- **Subtasks:** Support for nested subtasks with individual completion tracking.
- **Real-time Sync:** Live data synchronization across devices using Firestore real-time listeners.
- **Categorization:** Pre-defined categories (Work, Personal, Shopping, Health, Other).
- **Prioritization:** Task priority flags (High, Medium, Low).
- **Reminders:** Local device push notifications triggered at scheduled times (via Capacitor).
- **Recurrence:** Option to make tasks repeat (Daily, Weekly, Monthly) upon completion.
- **Search & Filtering:** Text-based search and category-based filtering.
- **Responsive UI:** 
  - Desktop: Persistent sidebar and horizontal stat cards.
  - Mobile: App-like layout with a hidden drawer, Floating Action Button (FAB), and a Bottom Sheet for task creation.
- **Theming:** Light and Dark mode toggle.

## 4. Key Functions in `app.js`

- `initAuth()`: Initializes the Firebase authentication state observer and manages the transition between the login screen and the main dashboard.
- `subscribeToTasks(uid)`: Attaches a real-time snapshot listener to the user's Firestore task collection, updating the local UI whenever changes occur in the cloud.
- `renderTasks()`: Clears the current DOM task list, filters the tasks based on the active category and search query, and iterates through the list to append task elements. It also updates the statistical counters.
- `createTaskElement(task)`: Generates the HTML markup and DOM node for an individual task, rendering the appropriate metadata pills (category, priority, reminders) and action buttons.
- `addTask()`: Reads inputs from the primary (desktop) add-task bar, sanitizes the data, and adds a new document to Firestore.
- `addSheetTask()`: Handles task creation specifically from the mobile Bottom Sheet UI, then saves to Firestore.
- `toggleTask(taskId, currentState)`: Flips the completion status of a task in Firestore. Also triggers the creation of a duplicate task if the original task has a recurrence rule set.
- `updateTask(taskId, updates)`: A utility function to push a partial object of updates to a specific Firestore document.
- `deleteTask(taskId)`: Deletes a specific task document from Firestore after confirming with the user.

## 5. Firebase Schema

**Collection Path:** `/users/{userId}/tasks/{taskId}`

**Document Structure:**
- `text` (String): The main task title.
- `notes` (String): Optional description or notes for the task.
- `category` (String): Identifier for the category (e.g., 'work', 'personal').
- `priority` (String): Priority level ('low', 'medium', 'high', 'none').
- `subtasks` (Array of Objects): List of subtasks. Each object contains `{ text: String, completed: Boolean, id: Number }`.
- `completed` (Boolean): Whether the task is checked off.
- `recurrence` (String): Repeating rule ('none', 'daily', 'weekly', 'monthly').
- `reminderTime` (String): ISO timestamp representing when the reminder should trigger.
- `notificationId` (Number): The numeric ID assigned to the Capacitor local notification, used for cancellation.
- `order` (Number): A timestamp used for default sorting.
- `createdAt` (Timestamp): Firestore server timestamp tracking when the task was created.

- `npm run build` - Compiles the web application for production into the `dist/` folder.
- `npm run preview` - Previews the compiled production build locally.

- `npm run open:ios` - Opens the iOS native project in Xcode.

## 9. Security Measures

- **Firestore Security Rules:** Access is strictly limited so that a user can only read, write, or delete documents within their own `/users/{userId}/...` path. Unauthenticated access is blocked.
- **XSS Prevention:** All user-generated text inputs (task titles, notes, subtasks) are sanitized using `DOMPurify` before being saved to the database and before being interpolated into the DOM via `innerHTML`.

## 10. Known Limitations

- **Recurrence Mechanism:** Task duplication upon completion relies on the client application executing the logic. If the network drops or the app is closed immediately, the recurring task might fail to generate.
- **Offline Support:** The app relies on Firestore's default offline caching. Advanced conflict resolution or robust offline-first background syncing is not heavily customized.
- **Notifications:** Push notifications are exclusively local device alarms handled by Capacitor. Remote push notifications (FCM) triggered by a server are not implemented.
- **Drag & Drop:** The UI has CSS classes to suggest drag-and-drop support, but persistent sorting logic backing up to the Firestore `order` field is not fully implemented across all filtered views.
